Lo que aprendí.

01

Conflicto entre señales de Django y el superusuario

Al resetear la base de datos y recrear el superusuario, la señal post-save asociada al modelo User intentaba crear automáticamente un perfil Employee para él, causando un error.

Lo resolví añadiendo una condición en el handler para excluir a los superusuarios del proceso de creación automática.

02

Integración de Chart.js para visualización de datos

Necesitaba mostrar estadísticas de asistencia de forma visual y nunca había usado ninguna librería de gráficos.

El reto fue aprender Chart.js desde cero e integrarlo con Django, pasando los datos desde las vistas del servidor a JavaScript de forma correcta. El resultado fueron gráficas interactivas integradas en los dashboards del sistema.

03

Gestión de señales de Django en el entorno de tests

Al ejecutar los tests, la señal post_save que crea automáticamente un perfil Employee al crear un User causaba conflictos: los tests fallaban porque intentaban crear un Employee para un usuario que ya lo tenía.

La solución fue desconectar la señal en setUpClass() y reconectarla en tearDownClass(), evitando efectos secundarios entre pruebas.
